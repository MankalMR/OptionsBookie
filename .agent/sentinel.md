## 2026-03-23 - Critical Supabase Client Misconfiguration | **Learning:** The standard Supabase client in `src/lib/database-secure.ts` was erroneously initialized with the `SUPABASE_SERVICE_ROLE_KEY`, which bypassed Row Level Security (RLS) for all regular user operations. RLS is critical for multitenant data isolation. | **Prevention:** Ensure the standard Supabase client is ALWAYS initialized with `NEXT_PUBLIC_SUPABASE_ANON_KEY`. Only use the service role key for dedicated admin clients (`supabaseAdmin`) or specific background jobs that require elevated privileges.
